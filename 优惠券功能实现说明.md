# 优惠券功能实现说明

## 📋 概述

已成功实现完整的优惠券系统，包括数据库表结构对应的实体类、数据访问层、业务逻辑层和API接口。

## 🗄️ 数据库表结构

### 1. coupon - 优惠券模板表
存储优惠券的基本信息和规则配置。

**主要字段：**
- `id`: 优惠券ID（主键）
- `code`: 优惠券码（唯一）
- `name`: 优惠券名称
- `type`: 类型（DISCOUNT折扣 / AMOUNT_OFF现金减免）
- `value`: 优惠值
- `min_amount`: 使用门槛金额
- `usage_limit_per_user`: 每个用户最多使用次数
- `total_limit`: 总发放数量限制
- `start_time` / `end_time`: 生效时间范围
- `is_enabled`: 是否启用

### 2. user_coupon - 用户优惠券领取表
记录用户实际领取的优惠券实例。

**主要字段：**
- `id`: 用户优惠券记录ID（主键）
- `coupon_id`: 关联优惠券模板
- `user_id`: 领取用户ID
- `source_type`: 领取来源（MANUAL/SYSTEM/CAMPAIGN/EXCHANGE/REGISTER/MEMBERSHIP）
- `receive_status`: 状态（RECEIVED/LOCKED/USED/EXPIRED/VOID）
- `received_at`: 领取时间
- `start_time` / `end_time`: 该用户券的实际生效/失效时间
- `used_at`: 使用时间
- `order_id`: 使用的订单ID

### 3. coupon_redemption - 优惠券核销记录表
记录优惠券的使用和抵扣详情。

**主要字段：**
- `id`: 核销记录ID（主键）
- `user_coupon_id`: 用户优惠券实例ID
- `coupon_id`: 优惠券ID（冗余）
- `user_id`: 用户ID
- `order_id`: 订单ID
- `original_amount`: 订单原始金额
- `discount_amount`: 优惠抵扣金额
- `final_amount`: 优惠后实付金额
- `status`: 核销状态（LOCKED/SUCCESS/CANCELLED/ROLLBACK）
- `redeemed_at`: 核销时间
- `rollback_at`: 回滚时间

## 🔌 API 接口

### 用户端接口 (/api/coupons)

#### 1. 查询可用优惠券列表
```
GET /api/coupons/available
```

#### 2. 根据优惠券码查询
```
GET /api/coupons/code/{code}
```

#### 3. 领取优惠券（兑换码）
```
POST /api/coupons/redeem
Body: {
  "couponCode": "TEST2026",
  "userId": 1
}
```

#### 4. 查询用户的优惠券
```
GET /api/coupons/user/{userId}
```

#### 5. 查询用户可用的优惠券
```
GET /api/coupons/user/{userId}/available
```

#### 6. 计算优惠后的金额
```
POST /api/coupons/calculate
Body: {
  "couponId": 1,
  "orderAmount": 150.00
}
```

#### 7. 验证优惠券是否可用
```
POST /api/coupons/validate
Body: {
  "couponId": 1,
  "orderAmount": 150.00
}
```

#### 8. 查询用户的核销记录
```
GET /api/coupons/user/{userId}/redemptions
```

#### 9. 统计用户优惠总金额
```
GET /api/coupons/user/{userId}/total-discount
```

## 💡 核心功能特性

### 1. 优惠券类型支持
- **折扣券 (DISCOUNT)**: 按百分比折扣，如打8折
- **现金减免券 (AMOUNT_OFF)**: 直接减免固定金额，如减20元

### 2. 优惠券状态流转
```
RECEIVED (已领取)
    ↓
LOCKED (锁定中 - 下单时)
    ↓
USED (已使用 - 支付成功)
    
或
    
LOCKED → RECEIVED (订单取消，释放优惠券)
USED → ROLLBACK (订单退款，恢复优惠券)
```

### 3. 领取来源追踪
- MANUAL: 后台手动发放
- SYSTEM: 系统自动发放
- CAMPAIGN: 活动领取
- EXCHANGE: 兑换码领取
- REGISTER: 注册赠送
- MEMBERSHIP: 会员赠送

### 4. 业务规则
- ✅ 优惠券启用/禁用控制
- ✅ 时间有效期验证
- ✅ 使用门槛验证（最低消费金额）
- ✅ 每人领取次数限制
- ✅ 总发放数量限制
- ✅ 优惠券锁定机制（防止重复使用）
- ✅ 核销记录审计追踪
- ✅ 支持回滚操作（退款场景）

### 5. 事务控制
所有关键操作都使用了 `@Transactional` 注解，确保数据一致性：
- 优惠券发放
- 优惠券锁定/释放
- 优惠券核销
- 核销回滚
