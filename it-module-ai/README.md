# IT Module AI

本文件记录 AI 模块的访问控制规则、任务执行模型和本轮回归范围，便于后续维护和二次验证。

## 知识库访问权限模型

- `owner` 和 `admin` 具备完整管理权限。
- `editor` 具备编辑知识库和文档的权限，但不能改 `ownerId`、`projectId`、`scopeType`。
- `viewer` 仅可读取授权范围内的数据。
- 私有知识库以成员关系为准，非成员不能读取、下载、查看任务，也不能发起取消。
- `pageByProject` 对普通用户只返回可访问的数据，管理员走全量查询。

## 文档级接口校验原则

- `documentId` 相关接口先做文档归属校验，再进入业务查询。
- 读取 chunks、下载文档、查看文档索引任务，都必须先通过 `requireDocumentRead`。
- 导入任务读取、取消、列表查询，都必须先通过对应知识库或任务级 guard。
- 控制器和 service 都保留校验，避免仅依赖单层防护。

## 导入 / 索引任务生命周期

- 创建索引任务或 ZIP 导入任务后，接口先返回任务对象，执行由后台 executor 接管。
- 任务初始状态为 `PENDING`，后台线程再尝试切换到 `RUNNING`。
- 任务只能沿允许路径流转到终态，重复 runner 执行会被状态机拦截。
- `cancel` 请求只对 `PENDING` / `RUNNING` 生效，终态任务直接返回当前状态。
