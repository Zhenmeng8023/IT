# 支付宝当面付配置指南

## 📋 申请步骤

### 1. 注册支付宝开发者账号
1. 访问 [支付宝开放平台](https://open.alipay.com/)
2. 使用支付宝账号登录
3. 完成开发者认证（需要实名认证）

### 2. 创建应用
1. 进入 [应用开发平台](https://console.open.alipay.com/)
2. 点击"创建应用"
3. 选择"网页/移动应用"
4. 填写应用信息：
   - 应用名称：你的项目名称
   - 应用图标：上传一个图标
   - 应用描述：简单描述
5. 提交审核（通常 1-2 个工作日）

### 3. 申请当面付功能
1. 应用审核通过后，进入应用详情
2. 点击"产品绑定" → "当面付"
3. 点击"申请开通"
4. 填写申请信息并提交

### 4. 获取配置参数
开通成功后，在应用详情页面获取以下参数：

- **APPID**：应用唯一标识
- **应用私钥**：自己生成的 RSA 私钥
- **支付宝公钥**：支付宝提供的公钥

## 🔧 配置参数

### 生成密钥对

方法 1：使用支付宝密钥生成工具
1. 下载 [密钥生成工具](https://opendocs.alipay.com/common/02kipl)
2. 运行工具，生成 RSA2(SHA256) 密钥
3. 保存私钥和公钥

方法 2：使用 OpenSSL 命令
```bash
# 生成私钥
openssl genrsa -out private_key.pem 2048

# 生成公钥
openssl rsa -in private_key.pem -pubout -out public_key.pem
```

### 修改 application.properties

打开 `it-main/src/main/resources/application.properties`，修改以下配置：

```properties
# 支付宝配置（替换为你的真实参数）
alipay.app-id=你的 APPID
alipay.private-key=你的应用私钥（不带换行符）
alipay.public-key=支付宝公钥（不带换行符）
```

**注意：**
- 私钥和公钥都要去掉换行符，连成一行
- 私钥是你自己生成的，公钥是支付宝提供的

## 🧪 沙箱环境 vs 生产环境

### 沙箱环境（推荐先测试）
- 用于开发测试，不会真实扣款
- 使用沙箱版支付宝 APP 扫码支付
- 配置使用沙箱环境的参数

### 生产环境
- 真实扣款
- 需要完成所有认证和申请
- 配置使用生产环境的参数

## 🎯 完整测试流程

### 1. 配置参数
在 `application.properties` 中填入你的支付宝参数

### 2. 启动后端
重新编译并运行 Spring Boot 应用

### 3. 测试支付
1. 打开前端页面：`http://localhost:3000/wallet`
2. 选择 VIP 套餐
3. 支付方式选择"支付宝"
4. 点击"立即开通"
5. 会自动打开新窗口显示支付宝支付页面
6. 使用支付宝 APP 扫码支付

### 4. 支付成功
- 支付成功后，会自动跳转回你的网站
- 后端会收到异步通知，更新订单状态
- 前端轮询检测到支付成功，更新 VIP 状态

## ⚠️ 注意事项

### 1. 密钥格式
- 私钥和公钥都要去掉所有换行符
- 保存为一行字符串
- 不要有空格

### 2. 回调地址
- 本地开发需要配置内网穿透（如 ngrok）
- 生产环境配置你的域名
- 回调地址格式：`https://your-domain.com/api/payment/callback/alipay`

### 3. 订单金额
- 单位：元
- 最多两位小数
- 如：90.00

### 4. 订单号
- 必须唯一
- 当前代码自动生成：ORDER + 时间戳 + 6 位随机数

## 💡 常见问题

### Q: 申请被拒绝怎么办？
A: 检查提交的资料是否完整，确保：
- 网站已备案
- 有明确的业务场景
- 应用描述清晰

### Q: 个人开发者能申请吗？
A: 可以，但需要：
- 完成实名认证
- 提供相关资质证明
- 说明使用场景

### Q: 费率是多少？
A: 当面付费率一般为 0.6%，具体以支付宝官方为准

### Q: 多久能到账？
A: 一般是 T+1 自动结算到支付宝账户
