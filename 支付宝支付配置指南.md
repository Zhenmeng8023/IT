## 🔧 沙箱环境配置

### 1. 获取沙箱应用信息

1. **访问支付宝开放平台**
   - 网址：https://open.alipay.com/develop/sandbox/app
   - 使用支付宝账号登录

2. **查看沙箱应用信息**
   - APPID：例如 `2021005217662499`
   - 应用网关：暂时不需要配置
   - 授权回调地址：暂时不需要配置

3. **生成密钥对**
   
   **方式一：使用支付宝密钥生成器（推荐）**
   - 下载地址：https://opendocs.alipay.com/common/02kipk
   - 选择 RSA2(SHA256) 密钥
   - 密钥长度：2048位
   - 点击"生成密钥"
   - 保存生成的**应用私钥**和**应用公钥**

   **方式二：使用 OpenSSL 命令生成**
   ```bash
   # 生成私钥
   openssl genrsa -out app_private_key.pem 2048
   
   # 生成公钥
   openssl rsa -in app_private_key.pem -pubout -out app_public_key.pem
   ```

4. **上传应用公钥到沙箱**
   - 在沙箱应用页面找到"RSA2(SHA256)密钥"
   - 点击"设置应用公钥"
   - 将生成的**应用公钥**内容粘贴进去（去掉首尾的 `-----BEGIN PUBLIC KEY-----` 和 `-----END PUBLIC KEY-----`）
   - 保存后，系统会生成**支付宝公钥**，复制保存

### 2. 配置文件修改

编辑 `it-main/src/main/resources/application.properties`：

```properties
# 支付宝配置（沙箱环境）
alipay.app-id=你的沙箱APPID
alipay.private-key=你的应用私钥（去除换行符，保持在一行）
alipay.public-key=支付宝公钥（从沙箱页面获取）
alipay.gateway-url=https://openapi.alipaydev.com/gateway.do
```

**重要提示：**
- 私钥必须去除所有换行符，保持在同一行
- 不要泄露私钥！
- 公钥可以公开

### 3. 获取沙箱测试账号

在沙箱页面底部可以找到：
- **买家账号**：用于测试支付（例如：tjx***@sandbox.com）
- **卖家账号**：用于接收款项（例如：qhb***@sandbox.com）
- **登录密码**：统一为 `111111`
- **支付密码**：统一为 `111111`

下载"沙箱版钱包"APP进行扫码支付测试。

## 🚀 生产环境配置

### 1. 创建正式应用

1. **访问支付宝开放平台**
   - 网址：https://open.alipay.com/platform/developerIndex.htm

2. **创建应用**
   - 点击"创建应用"
   - 填写应用名称、图标、描述等信息
   - 提交审核（通常需要1-3个工作日）

3. **签约支付产品**
   - 进入应用详情
   - 点击"产品绑定"
   - 选择需要的支付产品：
     - **电脑网站支付**：PC端网页支付
     - **当面付**：扫码支付
     - **手机网站支付**：移动端H5支付
   - 提交签约申请，等待审核

### 2. 配置密钥

与沙箱环境类似：
1. 生成 RSA2 密钥对
2. 上传应用公钥到正式应用
3. 获取支付宝公钥

### 3. 配置文件修改

```properties
# 支付宝配置（生产环境）
alipay.app-id=你的正式APPID
alipay.private-key=你的应用私钥（去除换行符）
alipay.public-key=支付宝公钥
alipay.gateway-url=https://openapi.alipay.com/gateway.do
```

### 4. 配置回调地址

确保以下地址可以公网访问：

```properties
# 回调域名（需要是公网可访问的地址）
payment.callback-domain=https://yourdomain.com
```

- **异步通知地址**：`${callback-domain}/api/payment/callback/alipay`
- **同步返回地址**：`${callback-domain}/payment/success`

## 🔑 密钥格式说明

### 正确的密钥格式

**应用私钥示例（去除换行符后）：**
```
MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQC7VJTUt9Us8cKjMzEfYyjiWA4R4/M2bS1+fWIcPm15j7A0+w5rSfwFQcKfVQR...（省略后续内容，全部在一行）
```

**支付宝公钥示例：**
```
MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEAsgDq4OvnUubZPRUCOgTIBhC5Y/b8eQP5NnJYl+gYxLBJ5d5Oa9...（省略后续内容，全部在一行）
```

### 错误的密钥格式 ❌

```
-----BEGIN PRIVATE KEY-----
MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQC7VJTUt9Us8cKj
MzEfYyjiWA4R4/M2bS1+fWIcPm15j7A0+w5rSfwFQcKfVQR...
-----END PRIVATE KEY-----
```

**错误原因：**
- 包含了 `-----BEGIN PRIVATE KEY-----` 标记
- 包含换行符

### 如何去除换行符

**方法一：手动处理**
1. 用文本编辑器打开密钥文件
2. 删除首尾的标记行
3. 删除所有换行符，使内容成为一行

**方法二：使用命令行（Linux/Mac）**
```bash
# 处理私钥
cat app_private_key.pem | grep -v "BEGIN\|END" | tr -d '\n' > private_key_oneline.txt

# 处理公钥
cat app_public_key.pem | grep -v "BEGIN\|END" | tr -d '\n' > public_key_oneline.txt
```

**方法三：使用 PowerShell（Windows）**
```powershell
# 处理私钥
(Get-Content app_private_key.pem) | Where-Object { $_ -notmatch "BEGIN|END" } | Out-String | ForEach-Object { $_.Replace("`n", "").Replace("`r", "") } | Set-Content private_key_oneline.txt
```

## ❓ 常见问题

### Q1: 提示 "InvalidKeyException: invalid key format"

**原因：** 密钥格式不正确，可能包含换行符或标记。

**解决：**
1. 检查私钥是否去除了所有换行符
2. 确认没有包含 `-----BEGIN PRIVATE KEY-----` 等标记
3. 重新生成密钥并正确配置

### Q2: 提示 "签名验证失败"

**原因：** 
- 私钥和公钥不匹配
- 使用了错误的支付宝公钥

**解决：**
1. 确认使用的是从支付宝平台下载的**支付宝公钥**，而不是你上传的应用公钥
2. 重新检查私钥配置是否正确
3. 确认签名类型为 RSA2

### Q3: 本地开发无法接收异步通知

**原因：** 支付宝无法访问 `localhost` 或 `127.0.0.1`

**解决方案：**

**方案一：使用内网穿透工具（推荐）**
```bash
# 使用 ngrok
ngrok http 18080

# 会得到一个公网地址，例如：https://abc123.ngrok.io
# 将此地址配置为 callback-domain
payment.callback-domain=https://abc123.ngrok.io
```

**方案二：部署到测试服务器**
- 将应用部署到有公网IP的服务器
- 配置域名指向该服务器

**方案三：暂时禁用异步通知（仅用于开发）**
```java
// PaymentOrderServiceImpl.java
if (isLocalEnvironment) {
    notifyUrl = ""; // 不设置异步通知
    returnUrl = "http://localhost:3000/payment/success?orderNo=" + orderNo;
}
```

> ⚠️ 注意：禁用异步通知后，只能依赖同步回调，可能存在订单状态不一致的风险。

### Q4: 沙箱支付时提示 "交易订单处理失败"

**原因：** 
- APPID 配置错误
- 使用了生产环境的密钥

**解决：**
1. 确认使用的是沙箱环境的 APPID
2. 确认使用的是沙箱环境生成的密钥
3. 确认网关地址是 `https://openapi.alipaydev.com/gateway.do`

### Q5: 如何切换沙箱和生产环境？

**方法一：使用不同的配置文件**

创建两个配置文件：
- `application-dev.properties`（沙箱环境）
- `application-prod.properties`（生产环境）

在启动时指定：
```bash
# 开发环境
java -jar app.jar --spring.profiles.active=dev

# 生产环境
java -jar app.jar --spring.profiles.active=prod
```

**方法二：使用环境变量**

```properties
alipay.app-id=${ALIPAY_APP_ID:2021005217662499}
alipay.private-key=${ALIPAY_PRIVATE_KEY:default-sandbox-key}
alipay.public-key=${ALIPAY_PUBLIC_KEY:default-sandbox-public-key}
alipay.gateway-url=${ALIPAY_GATEWAY_URL:https://openapi.alipaydev.com/gateway.do}
```

启动时传入环境变量：
```bash
export ALIPAY_APP_ID=你的生产APPID
export ALIPAY_PRIVATE_KEY=你的生产私钥
export ALIPAY_PUBLIC_KEY=你的生产公钥
export ALIPAY_GATEWAY_URL=https://openapi.alipay.com/gateway.do
java -jar app.jar
```
